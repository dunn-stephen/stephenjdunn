## Overview

Vocab Monster is built around a simple premise: language study tools fail when the loop is slow. The app keeps the repetition cycle tight, makes progress legible, and reduces friction around adding and reviewing terms.

## Technical Notes

- React and TypeScript drive the client application.
- Supabase handles persistence and authentication.
- The UI emphasizes quick keyboard-friendly repetition over dense feature menus.

## Challenges

The main constraint was not algorithmic difficulty. It was resisting feature sprawl and protecting the speed of the core review loop while still making the app feel encouraging rather than clinical.

## Outcome

The project serves as both a product experiment and a proving ground for interface decisions that prioritize daily use instead of one-time demo appeal.
